# Export & Sharing System — Design Spec

**Date:** 2026-04-15
**Approach:** Fully client-side (Approach A) — zero new dependencies

## Overview

Add a unified export/sharing system to DatabaseDiagram that lets users share diagrams with teammates and generate professional documentation. All exports happen in the browser using vanilla JS and built-in browser APIs. No new npm dependencies.

## Export Toolbar

A dropdown button labeled **Export / Share** added to the navigation bar on all three views (`dynamic-list.html`, `dynamic-visual.html`, `dynamic-analysis.html`).

**Dropdown options:**
1. Share Link
2. Download HTML
3. Download PDF
4. Download Markdown
5. Download PNG (visual view only)
6. Download SVG (visual view only)

The dropdown markup and JS logic is inlined in each page (matching the project's existing pattern of self-contained HTML files with inline code).

## Feature 1: Shareable URL

**Trigger:** User clicks "Share Link" in the export dropdown.

**Flow:**
1. Read current schema JSON from `sessionStorage` (`dbdiagram_data` key).
2. Compress using the browser's `CompressionStream` API (gzip).
3. Base64-encode the compressed bytes.
4. Construct URL: `{origin}/#data={base64}`.
5. Copy to clipboard via `navigator.clipboard.writeText()`.
6. Show a toast notification: "Link copied to clipboard!"

**On page load (index.html):**
1. Check `window.location.hash` for a `#data=` prefix.
2. If found, base64-decode and decompress via `DecompressionStream`.
3. Store the resulting schema JSON in `sessionStorage`.
4. Redirect to `dynamic-list.html`.

**Size estimates:**
- 50-table schema: ~30KB JSON → ~5-8KB gzip → ~7-11KB base64 in URL hash.
- Browser URL hash limits: most modern browsers support 2MB+ in the fragment.
- For schemas exceeding ~100KB compressed, show a warning toast suggesting HTML export instead.

**Fallback:** If `CompressionStream` is unavailable (older browsers), skip compression and encode the raw JSON as base64. URL will be longer but functional.

## Feature 2: Self-Contained HTML Export

**Trigger:** User clicks "Download HTML" in the export dropdown.

**Output:** A single standalone `.html` file that opens in any browser with no server or internet connection.

**What the exported file contains:**
- Embedded schema JSON as a `<script>` variable.
- A read-only table list viewer with:
  - Database name, version, and table count in the header.
  - Searchable table list with columns, types, keys, FK relationships.
  - Auto-categorization with color-coded category groups.
  - Category filter buttons.
  - Collapsible table cards (expand/collapse on click).
- All CSS inlined (dark theme, cards, grid layout).
- All JS inlined (categorization engine, rendering, search).
- Footer: "Generated by DatabaseDiagram on {date}".

**What it omits (to keep file size reasonable):**
- Interactive drag-and-drop visual diagram (too much JS complexity).
- Schema analysis page (user can run that in the full tool).
- Connection form / SQL import functionality.

**Generation approach:**
1. Build the complete HTML string using template literals in JS.
2. Create a `Blob` with type `text/html`.
3. Trigger download via a temporary `<a>` element with `URL.createObjectURL()`.

**Filename:** `schema-{database-name}-{date}.html`

**Expected file size:** ~40-80KB depending on schema size.

## Feature 3: PDF Export

**Trigger:** User clicks "Download PDF" in the export dropdown.

**Approach:** Open a print-optimized view in a new window and trigger `window.print()`. The browser's built-in print-to-PDF handles rendering.

**Print layout contents:**
1. **Cover section** — database name, version, table count, generation date.
2. **Table of contents** — all tables grouped by auto-categorized category.
3. **Table details** — each table as its own section:
   - Column table: name, type, nullable, PK/FK indicators.
   - Indexes list.
   - Foreign key relationships with referenced table and columns.
4. **Schema analysis summary** — letter grade and top findings (errors, warnings).
5. **Relationship summary** — compact table of all FK relationships across the schema.

**Print stylesheet (`@media print`) details:**
- White background, dark text (clean print appearance, no dark theme).
- Page breaks (`break-before: page`) before each category group.
- Compact table layouts that fit A4 and US Letter width.
- Interactive elements hidden (buttons, search bars, dropdowns).
- Repeated header on each page with database name.

**Flow:**
1. Build the print-optimized HTML layout as a string in JS (no new HTML file needed).
2. Open a new window via `window.open()` and write the HTML into it using `document.write()`.
3. The schema data is embedded directly into the generated HTML string.
4. Call `window.print()` automatically after the content renders.
5. Close the window after printing completes (or user cancels).

## Feature 4: Markdown Export

**Trigger:** User clicks "Download Markdown" in the export dropdown.

**Output:** A GitHub-Flavored Markdown file suitable for pasting into wikis (GitHub, Confluence, Notion).

**Document structure:**

```
# Database Schema: {database-name}
> Generated on {date} | {table-count} tables | {version}

## Table of Contents
- [{category-name}](#{category-anchor})
  - [{table-name}](#{table-anchor})

## {Category Name}

### {TableName}
| Column | Type | Nullable | Key |
|--------|------|----------|-----|
| {name} | {type} | {yes/no} | {PK/FK/} |

**Indexes:** {index-name} ({columns}) {unique?}

**Foreign Keys:**
- `{column}` -> `{ref_table}.{ref_column}` ({actions})

---

## Schema Analysis
**Grade: {letter}**
- {error-count} errors: {summary}
- {warning-count} warnings: {summary}
- {tip-count} tips
```

**Details:**
- Tables grouped by auto-categorized category (same grouping as list view).
- Standard GFM tables for column definitions.
- FK relationships as readable arrows (`->`).
- Analysis summary at the bottom with grade and top findings.

**Generation:** String concatenation from schema JSON, client-side.

**Filename:** `schema-{database-name}-{date}.md`

## Feature 5: Enhanced PNG Export

**Scope:** Visual diagram view only (extends existing canvas-based PNG export).

**Enhancements over current:**

1. **Footer bar** — rendered onto the canvas before export:
   - Database name, table count, generation date.
   - Subtle branding: "Generated by DatabaseDiagram".

2. **Embedded schema metadata** — schema JSON stored in the PNG's `tEXt` chunk:
   - After `canvas.toBlob()` produces the PNG binary, parse the PNG chunk structure.
   - Insert a `tEXt` chunk with keyword `DatabaseDiagram` and value being the JSON string.
   - Reconstruct the PNG binary and trigger the download.

**Re-import:** See Feature 7.

**Filename:** `schema-{database-name}-{date}.png`

## Feature 6: Enhanced SVG Export

**Scope:** Visual diagram view only (extends existing SVG export).

**Enhancements over current:**

1. **Footer bar** — a `<g>` element appended to the SVG with database name, table count, date.

2. **Embedded schema metadata** — schema JSON stored in a `<metadata>` element:
   ```xml
   <metadata>
     <dbdiagram:schema xmlns:dbdiagram="https://databasediagram.app/schema">
       {JSON string}
     </dbdiagram:schema>
   </metadata>
   ```

**Re-import:** See Feature 7.

**Filename:** `schema-{database-name}-{date}.svg`

## Feature 7: Image Re-Import

**Location:** `index.html` connection form.

**Current behavior:** The drag-and-drop zone accepts `.sql` files for SQL dump import.

**Enhancement:** Also accept `.png` and `.svg` files:

1. **PNG re-import:**
   - Read the file as `ArrayBuffer`.
   - Parse PNG chunks, look for a `tEXt` chunk with keyword `DatabaseDiagram`.
   - Extract the JSON value, parse it, store in `sessionStorage`.
   - Redirect to `dynamic-list.html`.

2. **SVG re-import:**
   - Read the file as text.
   - Parse as XML/DOM, look for `<metadata>` → `<dbdiagram:schema>` element.
   - Extract the JSON text content, parse it, store in `sessionStorage`.
   - Redirect to `dynamic-list.html`.

3. **Error handling:**
   - If the image doesn't contain embedded metadata, show an error: "This image doesn't contain DatabaseDiagram schema data. Only images exported from this tool can be re-imported."

**File input update:** The drag-and-drop zone's `accept` attribute and file type detection updated to include `.png` and `.svg` alongside `.sql`.

## UI Details

### Toast Notifications
A simple toast system for feedback messages (e.g., "Link copied!", "Downloading..."):
- Fixed position, bottom-center of viewport.
- Auto-dismiss after 3 seconds.
- Dark semi-transparent background, white text, subtle slide-up animation.

### Export Dropdown Styling
- Matches the existing dark theme and nav bar styling.
- Icons (optional unicode) next to each option for quick scanning.
- Hover highlight consistent with existing button styles.

## Technical Notes

**Browser APIs used:**
- `CompressionStream` / `DecompressionStream` — gzip for shareable URLs.
- `Blob` + `URL.createObjectURL()` — file downloads.
- `navigator.clipboard.writeText()` — copy link to clipboard.
- `window.print()` — PDF generation.
- `Canvas` API — PNG rendering (already in use).
- `DOMParser` — SVG metadata parsing for re-import.

**No new npm dependencies.** All functionality uses built-in browser APIs.

**Code organization:** Following the project's existing pattern, all code is inlined in each HTML file. The export dropdown markup and shared utility functions (toast, download helper, compression) are duplicated across the three view pages. This matches the existing approach where the categorization engine is already duplicated between `dynamic-list.html` and `dynamic-visual.html`.
